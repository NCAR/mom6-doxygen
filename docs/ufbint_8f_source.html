5C SUBPROGRAM: UFBINT
6C PRGMMR: WOOLLEN ORG: NP20 DATE: 1994-01-06
7C
8C ABSTRACT: THIS SUBROUTINE WRITES OR READS SPECIFIED VALUES TO OR FROM
9C THE CURRENT BUFR DATA SUBSET WITHIN INTERNAL ARRAYS, WITH THE
10C DIRECTION OF THE DATA TRANSFER DETERMINED BY THE CONTEXT OF
11C ABS(LUNIN) {I.E., IF ABS(LUNIN) POINTS TO A BUFR FILE THAT IS OPEN
12C FOR INPUT, THEN DATA VALUES ARE READ FROM THE INTERNAL DATA SUBSET;
13C OTHERWISE, DATA VALUES ARE WRITTEN TO THE INTERNAL DATA SUBSET}.
14C THE DATA VALUES CORRESPOND TO MNEMONICS WHICH ARE PART OF A
15C DELAYED-REPLICATION SEQUENCE, OR FOR WHICH THERE IS NO REPLICATION
16C AT ALL. IF UFBINT IS READING VALUES, THEN EITHER BUFR ARCHIVE
17C LIBRARY SUBROUTINE READSB OR READNS MUST HAVE BEEN PREVIOUSLY
18C CALLED TO READ THE SUBSET FROM UNIT ABS(LUNIN) INTO
19C INTERNAL MEMORY. IF IT IS WRITING VALUES, THEN EITHER BUFR ARCHIVE
20C LIBRARY SUBROUTINE OPENMG OR OPENMB MUST HAVE BEEN PREVIOUSLY
21C CALLED TO OPEN AND INITIALIZE A BUFR MESSAGE WITHIN MEMORY FOR THIS
22C ABS(LUNIN).

106C REMARKS:
107C 1) UFBINT CAN ALSO BE CALLED TO PROVIDE INFORMATION ABOUT A SINGLE

110C THE MNEMONIC IN STR MUST APPEAR AS IT DOES IN THE BUFR TABLE,

116C BRACKETING SYMBOLS).}
117C
118C FOR EXAMPLE:
119C
120C CALL UFBINT(LUNIN,PLEVL,1, 50,IRET,'{PLEVL}')
121C
122C WILL RETURN WITH IRET EQUAL TO THE NUMBER OF OCCURRENCES OF THE

125C OCCURRENCE OF PLEVL IN THE SUBSET. IF THERE ARE NO OCCURRENCES
126C OF PLEVL IN THE SUBSET, IRET IS RETURNED AS ZERO.
127C
128C 2) WHEN THE BUFR FILE IS OPEN FOR OUTPUT, UFBINT CAN BE USED TO
129C PRE-ALLOCATE SPACE FOR SOME OR ALL MNEMONICS WITHIN DELAYED
130C REPLICATION SEQUENCES. A SUBSEQUENT CALL TO BUFR ARCHIVE
131C LIBRARY ROUTINE UFBREP OR UFBSEQ THEN ACTUALLY STORES THE
132C VALUES INTO THE BUFR FILES. HERE ARE TWO EXAMPLES OF THIS:
133C
134C EXAMPLE 1) PROBLEM: AN OUTPUT SUBSET "SEQNCE" IS LAID OUT AS
135C FOLLOWS IN A BUFR TABLE (NOTE 16 CHARACTERS HAVE BEEN
136C REMOVED FROM THE LAST COLUMN TO ALLOW THE TABLE TO FIT IN
137C THIS DOCBLOCK):
138C
139C | SEQNCE | {PLEVL} |
140C | PLEVL | WSPD WDIR TSIG PRLC TSIG PRLC TSIG PRLC |
141C
142C -- OR --
143C
144C | SEQNCE | {PLEVL} |
145C | PLEVL | WSPD WDIR "PSEQ"3 |
146C | PSEQ | TSIG PRLC |
147C
148C IN THIS CASE THE APPLICATION PROGRAM MUST STORE VALUES WHICH
149C HAVE STANDARD REPLICATION NESTED INSIDE OF A DELAYED
150C REPLICATION SEQUENCE. FOR EXAMPLE, ASSUME 50 LEVELS OF WIND
151C SPEED, WIND DIRECTION, OBSERVED PRESSURE, FIRST GUESS
152c PRESSURE AND ANALYZED PRESSURE ARE TO BE WRITTEN TO "SEQNCE".
153C
154C THE FOLLOWING LOGIC WOULD ENCODE VALUES PROPERLY:
155C.....................................................................
156C ....
157C REAL*8 DROBS(2,50)
158C REAL*8 SROBS(2,150)
159C ....
160C DO I=1,50
161C DROBS(1,I) = Value of wind speed on level "I"
162C DROBS(2,I) = Value of wind direction on level "I"
163C SROBS(1,I*3-2) = Value of observed pressure on level "I"
164C SROBS(2,I*3-2) = 25. ! Value in Code Table 0-08-021 (TSIG)
165C ! for time sigificance (Nominal
166C ! reporting time) for observed
167C ! pressure on level "I"
168C SROBS(1,I*3-1) = Value of first guess pressure on level "I"
169C SROBS(2,I*3-1) = 27. ! Value in Code Table 0-08-021 (TSIG)
170C ! for time sigificance (First guess)
171C ! for first guess pressure on level "I"
172C SROBS(1,I*3) = Value of analyzed pressure on level "I"
173C SROBS(2,I*3) = 16. ! Value in Code Table 0-08-021 (TSIG)
174C ! for time sigificance (Analysis) for
175C ! analyzed pressure on level "I"
176C ENDDO
177C
178C ! The call to UFBINT here will not only store the 50
179C ! values of WSPD and WDIR into the BUFR subset, it
180C ! will also allocate the space to store three
181C ! replications of TSIG and PRLC on each of the 50
182C ! delayed-replication "levels"
183C CALL UFBINT(LUNIN,DROBS,2, 50,IRET,'WSPD WDIR')

185C ! The call to UFBREP here will actually store the 150
186C ! values of both TSIG and PRLC (three values for each
187C ! on 50 delayed-replication "levels")

199C EXAMPLE 2) PROBLEM: AN INPUT SUBSET, "REPT_IN", AND AN OUTPUT
200C SUBSET "REPT_OUT", ARE LAID OUT AS FOLLOWS IN A BUFR TABLE
201C (NOTE 16 CHARACTERS HAVE BEEN REMOVED FROM THE LAST COLUMN

206C | PLEVL | PRLC TMBD REHU WDIR WSPD
207C
208C IN THIS CASE THE APPLICATION PROGRAM IS READING IN VALUES
209C FROM A BUFR FILE CONTAINING SUBSET "REPT_IN", CONVERTING
210C MONTH AND DAY TO DAY OF YEAR, AND THEN WRITING VALUES TO
211C SUBSET "REPT_OUT" IN ANOTHER BUFR FILE. A CONVENIENT WAY TO
212C DO THIS IS TO CALL UFBSEQ TO READ IN AND WRITE OUT THE
213C VALUES, HOWEVER THIS IS COMPLICATED BY THE PRESENCE OF THE
214C DELAYED-RELICATION SEQUENCE "PLEVL" BECAUSE THE OUTPUT CALL
215C TO UFBSEQ DOES NOT KNOW A-PRIORI HOW MANY REPLICATIONS ARE
216C NEEDED TO STORE THE CONTENTS OF "PLEVL" (IT SETS THE NUMBER
217C TO ZERO BY DEFUALT). A CALL TO UFBINT IS FIRST NEEDED TO
218C ALLOCATE THE SPACE AND DETERMINE THE NUMBER OF LEVELS NEEDED
219C TO STORE ALL VALUES IN "PLEVL".
220C
221C THE FOLLOWING LOGIC WOULD PEFORM THE READ/WRITE PROPERLY:
222C.....................................................................
223C ....
224C REAL*8 OBSI(2000),OBSO(1999),PLEVL(5,255),REPS_8
225C CHARACTER*8 SUBSET
226C ....

228C CALL DATELEN(10)

230C ! Open input BUFR file in LUBFI and open output BUFR file in
231C ! LUBFJ, both use the BUFR table in LINDX
232C CALL OPENBF(LUBFI,'IN', LINDX)
233C CALL OPENBF(LUBFJ,'OUT',LINDX)
234C
235C ! Read through the BUFR messages in the input file
236C DO WHILE(IREADMG(LUBFI,SUBSET,IDATE).EQ.0)
237C
238C ! Open message (for writing) in output file
239C CALL OPENMB(LUBFJ,'REPT_OUT',IDATE)
240C
241C ! Read through the subsets in this input BUFR messages
242C DO WHILE(IREADSB(LUBFI).EQ.0)
243C
244C ! This call to UFBSEQ will read in the entire contents
245C ! of subset "REPT_IN", storing them into array OBSI
246C ! (Note: On input, UFBSEQ knows how many replications
247C of "PLEV" are present)
248C CALL UFBSEQ(LUBFI,OBSI,2000,1,IRET,'REPT_IN')
249C
250C ! This call to UFBINT will return the number of
251C ! replications ("levels") in "PLEVL" for subset
252C ! "REPT_IN"" ! {see 1) above in REMARKS}
253C CALL UFBINT(LUBFI,REPS_8,1,1,IRET,'{PLEVL}')
254C IREPS = REPS_8
255C
256C IYR = OBSI(1)
257C IMO = OBSI(2)
258C IDA = OBSI(3)
259C CALL xxxx(IYR, IMO, IDA, JDY) ! convert month and day
260C ! to day of year (JDY)
261C OBSO(1) = OBSI(1)
262C OBSO(2) = JDY
263C DO I = 3,1999
264C OBSO(I) = OBSI(1+1)
265C ENDDO
266C
267C PLEVL = 10E10
268C
269C ! The call to UFBINT here will allocate the space to
270C ! later allow UFBSEQ to store IREPS replications of
271C ! "PLEVL" into the output BUFR subset "REPT_OUT" (note
272C ! here it is simply storing missing values)
273C CALL UFBINT(LUBFJ,PLEVL,5,IREPS,IRET,
274C $ 'PRLC TMBD REHU WDIR WSPD')
275C
276C ! The call to UFBSEQ here will write out the entire
277C ! contents of subset "REPT_OUT", reading them from
278C ! array OBSO
279C CALL UFBSEQ(LUBFJ,OBSO,1999,1,IRET,'REPT_OUT')
280C
281C ! Write the subset into the output BUFR message
282C CALL WRITSB(LUBFJ)
283C ENDDO
284C
285C ! All done
