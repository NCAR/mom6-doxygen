Detailed Description

Filenames for state vector IO Aim is to store the io filenames for state read/writes.

  • Restarts
  • Diagnostics
  • Inflation files

Usage: A file_info_type is created with a call to file_info_type = io_filenames_init()

The file_info_type is public and contains:

  • IO file names
  • IO metadata such as units, copy number, description, ...

The restart_names_types contain a 2D array of filenames (num files, num_domains).

The file_info_type is passed to the state IO routines: read_state, write_state, and filter_state_space_diagnostics (diagnostic file)

The internals of the file_info_type are accessed through the accessor functions listed below. assert_file_info_initialized() and assert_restart_names_initialized() should be used to check that the file_info_type has been initialized before attempting any IO.

Diagnostic files could have different netcdf variable ids
