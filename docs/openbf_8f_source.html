3C$$$ SUBPROGRAM DOCUMENTATION BLOCK
4C
5C SUBPROGRAM: OPENBF
6C PRGMMR: WOOLLEN ORG: NP20 DATE: 1994-01-06
7C
8C ABSTRACT: THIS SUBROUTINE NORMALLY (I.E. EXCEPT WHEN INPUT ARGUMENT
9C IO IS 'QUIET') IDENTIFIES A NEW LOGICAL UNIT TO THE BUFR ARCHIVE
10C LIBRARY SOFTWARE FOR INPUT OR OUTPUT OPERATIONS. HOWEVER, THE
11C FIRST TIME IT IS CALLED, IT ALSO FIGURES OUT SOME IMPORTANT
12C INFORMATION ABOUT THE LOCAL MACHINE ON WHICH THE SOFTWARE IS BEING
13C RUN (VIA A CALL TO BUFR ARCHIVE LIBRARY SUBROUTINE WRDLEN), AND IT
14C ALSO INITIALIZES ARRAYS IN MANY BUFR ARCHIVE LIBRARY COMMON BLOCKS
15C (VIA A CALL TO BUFR ARCHIVE LIBRARY SUBROUTINE BFRINI). UP TO 32
16C LOGICAL UNITS CAN BE CONNECTED TO THE BUFR ARCHIVE LIBRARY SOFTWARE
17C AT ANY ONE TIME.
18C
19C NOTE: IF IO IS PASSED IN AS 'QUIET', THEN OPENBF PERFORMS ONLY ONE
20C FUNCTION - IT SIMPLY SETS THE "DEGREE OF PRINTOUT" SWITCH IPRT (IN
21C COMMON BLOCK /QUIET/) TO THE VALUE OF INPUT ARGUMENT LUNDX,
22C OVERRIDING ITS PREVIOUS VALUE. A DEFAULT IPRT VALUE OF 0 (I.E.
23C "LIMITED PRINTOUT") IS SET DURING THE FIRST CALL TO THIS ROUTINE,
24C BUT THIS OR ANY OTHER IPRT VALUE MAY BE SET AND RESET AS OFTEN AS
25C DESIRED VIA SUCCESSIVE CALLS TO OPENBF WITH IO = 'QUIET'.
26C IN ALL SUCH CASES, OPENBF SIMPLY (RE)SETS IPRT AND THEN RETURNS
27C WITHOUT ACTUALLY OPENING ANY FILES. THE DEGREE OF PRINTOUT
28C INCREASES AS IPRT INCREASES FROM "-1" TO "0" TO "1" TO "2".
29C
30C PROGRAM HISTORY LOG:
31C 1994-01-06 J. WOOLLEN -- ORIGINAL AUTHOR
32C 1998-07-08 J. WOOLLEN -- REPLACED CALL TO CRAY LIBRARY ROUTINE
33C "ABORT" WITH CALL TO NEW INTERNAL BUFRLIB
34C ROUTINE "BORT"
35C 1999-11-18 J. WOOLLEN -- THE NUMBER OF BUFR FILES WHICH CAN BE

38C BUFR FILES UNDER THE MPI)

46C DIAGNOSTIC INFO WHEN ROUTINE TERMINATES
47C ABNORMALLY, UNUSUAL THINGS HAPPEN OR FOR
48C INFORMATIONAL PURPOSES
49C 2004-08-18 J. ATOR -- ADDED SAVE FOR IFIRST FLAG AND IO="NODX"

54C USAGE: CALL OPENBF (LUNIT, IO, LUNDX)
55C INPUT ARGUMENT LIST:
56C LUNIT - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR BUFR FILE
57C (UNLESS IO IS 'QUIET', THEN A DUMMY)
58C IO - CHARACTER*(*): FLAG INDICATING HOW LUNIT IS TO BE
59C USED BY THE SOFTWARE:
60C 'IN' = input operations
61C 'OUT' = output operations
62C 'APN' = same as 'OUT', except begin writing at end
63C of file ("append")
64C 'APX' = same as 'APN', except backspace before
65C appending
66C 'NODX' = same as 'OUT', except don't write dictionary
67C (i.e. DX) table messages to LUNIT
68C 'NUL' = same as 'OUT', except don't write any
69C messages whatsoever to LUNIT (e.g. when
70C subroutine WRITSA or WRITCA is to be used)
71C 'QUIET' = LUNIT is ignored, this is an indicator
72C that the value for IPRT in COMMON block
73C /QUIET/ is being reset (see LUNDX)
74C LUNDX - INTEGER: IF IO IS NOT 'QUIET':
75C FORTRAN logical unit number containing
76C dictionary table information to be used in
77C reading/writing from/to LUNIT (depending
78C on the case); may be set equal to LUNIT if
79C dictionary table information is already
80C embedded in LUNIT
81C IF IO IS 'QUIET':
82C Indicator for degree of printout:
83C -1 = NO printout except for ABORT
84C messages
85C 0 = LIMITED printout (default)
86C 1 = ALL warning messages are printed
87C out
88C 2 = ALL warning AND informational
89C messages are printed out
90C (Note: this does not change until OPENBF
91C is again called with IO equal to

94C INPUT FILES:
