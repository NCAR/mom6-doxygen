Detailed Description

Default routines for netCDF reading and writing dart model time. If your model uses a different name for the time dimension or has a different way of handing/storing time, it must provide a custom read_model_time() and write_model_time() routine.

Make a stab at reading the time from the input file and setting the calendar. This routine assumes the file hasn't been opened yet, and closes it when it's done.

assumes time variable is real and fractional days. if this isn't true, user has to supply their own read time routine.
