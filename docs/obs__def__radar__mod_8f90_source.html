6!-----------------------------------------------------------------------------
7! DART radar observation module, including the observation operators for the
8! two primary radar-observation types -- Doppler velocity and reflectivity --
9! plus other utility subroutines and functions. A number of simplifications
10! are employed for the observation operators. Most notably, the model state
11! is mapped to a "point" observation, whereas a real radar observation is a
12! volumetric sample. The implications of this approximation have not been
13! investigated fully, so in the future it might be worth developing and testing
14! more sophisticated observation operators that produce volumetric power-
15! weighted samples.
16!
17! This module is able to compute reflectivity and precipitation fall speed
18! (needed for computing Doppler radial velocity) from the prognostic model
19! fields only for simple single-moment microphysics schemes such as the Kessler
20! and Lin schemes. If a more complicated microphysics scheme is used, then
21! reflectivity and fall speed must be accessible instead as diagnostic fields
22! in the model state.

204!-------------------------------------------------------------
205! Namelist with default values
206!
207! Obsolete: convert_to_dbz and dbz_threshold
208! convert_to_dbz and dbz_threshold have both been removed from the namelist.
209! Values will always be converted to dBZ, and threshold was only used to
210! ensure the log() call never saw a real 0.0_r8. Please remove these
211! values from your namelist to avoid a run-time error.
212!
213! There are two replicated sets of 3 namelist values below.
214! In each case, there is 1 logical value and 2 numeric values.
215! If the logical is false, the numeric values are ignored.
216! If true, then the 2 numeric values are:
217! 1) a threshold value. If the observation or forward operator value is
218! less than this threshold, it will be set to a different value.
219! 2) the value it should be set to.
220! The value is separate from the threshold to allow, for example, the option
221! of setting all values below -20 dBZ to -40 dBZ.
222!
223! If the observation value or the forward operator already has a value of
224! missing_r8 it is assumed either the istatus is marked as failed (for the
225! forward operator) or that the QC (quality control) flag is set to not
226! assimilate this observation and that value is left unchanged regardless of
227! the setting on the apply_ref_limit flag. Note however that it is not a
228! good idea to reset a good but small observation value to missing_r8 -- do
229! not use it as the lowest_reflectivity setting.
230!
231! The next 3 namelist items apply to the incoming observation values.
232! They are in the namelist so they can be changed at runtime, instead
233! of set only when the observation file is originally generated.
234!
235! apply_ref_limit_to_obs:
236! Logical. If .TRUE. replace all reflectivity values less than
237! "reflectivity_limit_obs" with "lowest_reflectivity_obs" value.
238! If .FALSE. leave all values as-is. Defaults to .FALSE.
239!
240! reflectivity_limit_obs:
241! The threshold value. Observed reflectivity values less than this
242! threshold will be set to the "lowest_reflectivity_obs" value.
243! Units are dBZ. Defaults to -10.0.
244!
245! lowest_reflectivity_obs:
246! The 'set-to' value. Observed reflectivity values less than the
247! threshold will be set to this value. Units are dBZ. Defaults to -10.0.
248!
249! The next 3 namelist items apply to the forward operator values (the returned
250! value from each ensemble member predicting what the observation value should
251! be given the current state in this particular member).
252!
253! apply_ref_limit_to_fwd_op:
254! Same as "apply_ref_limit_to_obs", but for the forward operator.
255!
256! reflectivity_limit_fwd_op:
257! Same as "reflectivity_limit_obs", but for the forward operator values.
258!
259! lowest_reflectivity_fwd_op:
260! Same as "lowest_reflectivity_obs", but for the forward operator values.
261!
262! max_radial_vel_obs:
263! Integer value. Maximum number of observations of this type to support at
264! run time. This is combined total of all obs_seq files, for example the
265! observation diagnostic program potentially opens multiple obs_seq.final
266! files, or the obs merge program can also open multiple obs files.
267! Default is 1,000,000 observations.
268!
269! dielectric_factor:
270! According to Smith (1984), there are two choices for the dielectric
271! factor, depending on how the snow particle sizes are specified.
272! If melted raindrop diameters are used, then the factor is 0.224. If
273! equivalent ice sphere diameters are used, then the factor is 0.189.
274! Since the common convention is to use melted raindrop diameters, the
275! default here is 0.224.
276!
277! n0_rain, n0_graupel, n0_snow:
278! Intercept parameters (m^-4) for size distributions of each hydrometeor.
279! The defaults below are for the Lin et al. microphysics scheme
280! with the Hobbs settings for graupel/hail. (The Hobbs graupel settings
281! are also the default for the Lin scheme in WRF 2.2 and 3.0.)
282!
283! rho_rain, rho_graupel, rho_snow:
284! Density (kg m^-3) of each hydrometeor type. The defaults below are for the
285! Lin et al. microphysics scheme with the Hobbs setting for graupel/hail.
286!
287! allow_wet_graupel:
288! Logical. It is difficult to predict/diagnose whether graupel/hail has a
289! wet or dry surface. Even when the temperature is above freezing,
290! evaporation and/or absorption can still result in a dry surface. This
291! issue is important because the reflectivity from graupel with a wet surface
292! is significantly greater than that from graupel with a dry surface.
293! Currently, the user has two options for how to compute graupel
294! reflectivity. If allow_wet_graupel is .false. (the default), then graupel
295! is always assumed to be dry. If allow_wet_graupel is .true., then graupel
296! is assumed to be wet (dry) when the temperature is above (below) freezing.
297! A consequence is that a sharp gradient in reflectivity will be produced at
298! the freezing level. In the future, it might be better to provide the
299! option of having a transition layer.
300!
301! microphysics_type:
302! Integer. Tells obs_def_radar_mod what microphysical scheme is employed
303! to enable some smarter decisions in handling fall velocity and radar
304! reflectivity.
305! 1 - Kessler scheme
306! 2 - Lin et al. microphysics (default)
307! 3 - User selected scheme where 10 cm reflectivity and power weighted fall
308! velocity are expected in the state vector (failure if not found)
309! 4 - User selected scheme where only power weighted fall velocity is
310! expected (failure if not found)
311! 5 - User selected scheme where only reflectivity is expected (failure
312! if not found)
313! -1 - ASSUME FALL VELOCITY IS ZERO, allows over-riding the failure modes
314! above if reflectivity and/or fall velocity are not available but a
315! result is desired for testing purposes only.
316!
317! allow_dbztowt_conv:
318! Logical. Flag to enable use of the dbztowt routine where reflectivity is
319! available, but not the power-weighted fall velocity. This scheme uses
320! emperical relations between reflectivity and fall velocity, with poor
321! accuracy for highly reflective, low density particles (such as water coated
322! snow aggregates). Expect questionable accuracy in radial velocity from
323! the forward operator with high elevation angles where ice is present in
324! the model state.
325

380subroutine initialize_module
381
382! Called once to set values and allocate space
383
384integer :: iunit, io, rc
385
386! Prevent multiple calls from executing this code more than once.
387if (module_initialized) return
388

393
394! Read the namelist entry.

440subroutine read_radial_vel(velkey, ifile, fform)
441
442! Main read subroutine for the radial velocity observation auxiliary data.
443
444integer, intent(out) :: velkey
445integer, intent(in) :: ifile
446character(len=*), intent(in), optional :: fform
447
448character(len=8) :: header
449logical :: is_asciifile
450type(location_type) :: radar_location
451real(r8) :: beam_direction(3)
452real(r8) :: nyquist_velocity
453integer :: oldkey
454
455! radar_location: Refers to the lat/lon/height/vertical coordinate option
456! for the radar. Its type is defined through location_type in location_mod.
457! Uses the read_location function in location_mod.
458!
459! beam_direction: This is a 3-element array specific to the radar module:
460! beam_direction(1) = sin(azimuth)*cos(elevation)
461! beam_direction(2) = cos(azimuth)*cos(elevation)
462! beam_direction(3) = sin(elevation)
463
464if ( .not. module_initialized ) call initialize_module
465
466is_asciifile = ascii_file_format(fform)
467
468if (is_asciifile) then
469 ! Read the character identifier for verbose formatted output
470 read(ifile, fmt="(a8)") header
471 if(header /= 'platform') then
472 call error_handler(e_err,'read_radial_vel', &
473 "Expected location header 'platform' in input file", &
474 source, revision, revdate)
475 endif
476endif
477
478! read_location is a DART library routine that expects an optional string
479! arg, but the other two read routines are local to this module and we can
480! tell them exactly what format to be reading because we already know it.
481radar_location = read_location(ifile, fform)
482beam_direction = read_beam_direction(ifile, is_asciifile)
483nyquist_velocity = read_nyquist_velocity(ifile, is_asciifile)
484
485! Read in the velkey for this particular observation, however, it will
486! be discarded and a new, unique key will be generated in the set routine.
487if (is_asciifile) then
488 read(ifile, *) oldkey
489else
490 read(ifile) oldkey
491endif
492
493! Generate new unique radial velocity observation key, and set the contents
494! of the private defined type.
495call set_radial_vel(velkey, radar_location, beam_direction, nyquist_velocity)
496
497end subroutine read_radial_vel

501subroutine write_radial_vel(velkey, ifile, fform)
502
503! Write radial velocity auxiliary information to the obs_seq file.
504
505integer, intent(in) :: velkey, ifile
506character(len=*), intent(in), optional :: fform
507
508logical :: is_asciifile
509type(location_type) :: radar_location
510real(r8) :: beam_direction(3)
511real(r8) :: nyquist_velocity
512
513if ( .not. module_initialized ) call initialize_module
514
515! Simple error check on key number before accessing the array
516call velkey_out_of_range(velkey)
517
518is_asciifile = ascii_file_format(fform)
519
520if (is_asciifile) then
521 ! Write the 5 character identifier for verbose formatted output
522 write(ifile, "('platform')")
523endif
524
525! Extract the values for this key and call the appropriate write routines.
526radar_location = radial_vel_data(velkey)%radar_location
527beam_direction(:) = radial_vel_data(velkey)%beam_direction(:)
528nyquist_velocity = radial_vel_data(velkey)%nyquist_velocity
529
530! write_location routine is part of the DART library and wants the optional
531! format string argument. The other two routines are local to this module,
532! and we have already figured out if it is a unformatted/binary file or
533! formatted/ascii, so go ahead and pass that info directly down to the routines.
534call write_location(ifile, radar_location, fform)
535call write_beam_direction(ifile, beam_direction(:), is_asciifile)
536call write_nyquist_velocity(ifile, nyquist_velocity, is_asciifile)
537
538! Write out the velkey used for this run, however this will be discarded
539! when this observation is read in and a new key will be generated.
540! (It may be useful for correlating error messages or identifying particular
541! observations so we are leaving it as part of the aux data.)
542if (is_asciifile) then
543 write(ifile, *) velkey
544else
545 write(ifile) velkey
546endif
547
548end subroutine write_radial_vel

552function read_beam_direction(ifile, is_asciiformat)
553
554! Reads beam_direction from file that was written by write_beam_direction.
555! See read_radial_vel for additional discussion.
556
557integer, intent(in) :: ifile
558logical, intent(in) :: is_asciiformat
559real(r8) :: read_beam_direction(3)
560
561character(len=5) :: header
562real(r8) :: beam_direction(3)
563
564if ( .not. module_initialized ) call initialize_module
565
566
567if (is_asciiformat) then
568 read(ifile, "(a5)" ) header
569
570 if(header /= 'dir3d') then
571 write(msgstring,*)"Expected beam_direction header 'dir3d' in input file, got ", header
572 call error_handler(e_err, 'read_beam_direction', msgstring, source, revision, revdate)
573 endif
574 ! Now read the beam_direction data value into temporaries
575 read(ifile, *) beam_direction(1), beam_direction(2), beam_direction(3)
576else
577 ! No header label, just the binary direction values.
578 read(ifile) beam_direction(1), beam_direction(2), beam_direction(3)
579endif
580
581! Check for illegal values

842
843subroutine get_expected_radial_vel(state_handle, ens_size, location, velkey, &
844 radial_vel, istatus)
845
846! This is the main forward operator routine for radar Doppler velocity.
847
848type(ensemble_type), intent(in) :: state_handle
849integer, intent(in) :: ens_size
850type(location_type), intent(in) :: location
851integer, intent(in) :: velkey
852real(r8), intent(out) :: radial_vel(ens_size)
853integer, intent(out) :: istatus(ens_size)
854
855! Given a location and the state vector from one of the ensemble members,
856! compute the model-predicted radial velocity that would be observed
857! at that location. The value is returned in 'radial_vel'.
858! 'istatus' is the return code. 0 is success; any positive value signals an
859! error (different values can be used to indicate different error types).
860! Negative istatus values are reserved for internal use only by DART.
861!
862! The along-beam component of the 3-d air velocity is computed from the
863! u, v, and w fields plus the beam_direction. The along-beam component
864! of power-weighted precipitation fall velocity is added to the result.
865!
866! Reference: Lin et al., 1983 (J. Climate Appl.Meteor., 1065-1092)
867
868real(r8), dimension(ens_size) :: u, v, w, precip_fall_speed
869integer, dimension(ens_size) :: u_istatus, v_istatus, w_istatus, p_istatus
870logical :: return_now
871real(r8) :: debug_location(3)
872logical :: debug = .false. ! set to .true. to enable debug printout
873
874if ( .not. module_initialized ) call initialize_module
875
876! Simple error check on key number before accessing the array
877call velkey_out_of_range(velkey)
878istatus(:)= 0
879
880call interpolate(state_handle, ens_size, location, qty_u_wind_component, u, u_istatus)
881call track_status(ens_size, u_istatus, radial_vel, istatus, return_now)
882if (return_now) return
883
884call interpolate(state_handle, ens_size, location, qty_v_wind_component, v, v_istatus)
885call track_status(ens_size, v_istatus, radial_vel, istatus, return_now)
886if (return_now) return
887
888
889call interpolate(state_handle, ens_size, location, qty_vertical_velocity, w, w_istatus)
890call track_status(ens_size, w_istatus, radial_vel, istatus, return_now)
891if (return_now) return
892
893
894call get_expected_fall_velocity(state_handle, ens_size, location, precip_fall_speed, p_istatus)
895call track_status(ens_size, p_istatus, radial_vel, istatus, return_now)
896if (return_now) return
897
898where (istatus == 0)
899
900 radial_vel = radial_vel_data(velkey)%beam_direction(1) * u + &
901 radial_vel_data(velkey)%beam_direction(2) * v + &
902 radial_vel_data(velkey)%beam_direction(3) * (w - precip_fall_speed)
903
904end where
905
906if (debug) then
907 debug_location = get_location(location)
908 print *
909 print *, 'radial velocity key: ', velkey
910 print *, 'obs location (deg): ', debug_location(1), &
911 debug_location(2), debug_location(3)
912 print *, 'obs location (rad): ', debug_location(1)*deg2rad, &
913 debug_location(2)*deg2rad, debug_location(3)
914 print *, 'interpolated u: ', u
915 print *, 'interpolated v: ', v
916 print *, 'interpolated w: ', w
917 print *, 'interp or derived fall speed: ', precip_fall_speed
918 print *, 'final radial_vel: ', radial_vel
919 print *, 'istatus: ', istatus
920endif
921
922end subroutine get_expected_radial_vel

925!----------------------------------------------------------------------
926! Expected fall velocity section
927!----------------------------------------------------------------------
928!----------------------------------------------------------------------
929
930subroutine get_expected_fall_velocity(state_handle, &
931 ens_size, location, precip_fall_speed, istatus)
932
933! This is the main forward operator routine for the expected
934! fall velocity, and it also used as part of computing expected
935! radial velocity.
936
937type(ensemble_type), intent(in) :: state_handle
938integer, intent(in) :: ens_size
939type(location_type), intent(in) :: location
940real(r8), intent(out) :: precip_fall_speed(ens_size)
941integer, intent(out) :: istatus(ens_size)
942
943! Local vars
944logical, save :: first_time = .true.
945real(r8), dimension(ens_size) :: qr, qg, qs, rho, temp, refl
946integer, dimension(ens_size) :: qr_istatus, qg_istatus, qs_istatus, rho_istatus, refl_istatus, temp_istatus
947integer :: imem
948logical :: return_now
949! If the model can return the precipitation fall speed directly, let it do
950! so. Otherwise, attempt to compute if Kessler or Lin type microphysics,
951! or see if the dbztowt routine is desired. Note that the computation for
952! Lin and Kessler is expected to be reasonably accurate for 10cm radar.
953
954! Should we check microphysics_type var or just go ahead and try to get a value?
955
956istatus(:) = 0
957precip_fall_speed(:) = 0.0_r8
958
959call interpolate(state_handle, ens_size, location, qty_power_weighted_fall_speed, &
960 precip_fall_speed, istatus)
961
962! If able to get value, QTY_POWER_WEIGHT_FALL_SPEED is the
963! the state so you can return here.
964if (any(istatus == 0) ) return
965
966! If the user explicitly wanted to interpolate in the field, try to complain
967! if it could not. Note that the interp could fail for other reasons.
968if (microphysics_type == 3 .or. microphysics_type == 4) then
969 ! Could return a specific istatus code here to indicate this condition.
970 if (first_time) then
971 call error_handler(e_msg,'get_expected_fall_velocity', &
972 'interpolate failed. Fall Speed may NOT be in state vector', '', '', '')
973 first_time = .false.
974 endif
975 return
976endif
977
978! QTY_POWER_WEIGHT_FALL_SPEED is not in the state, try to interpolate
979! in a different way
980istatus(:) = 0
981
982! If not in the state vector, try to calculate it here based on the
983! setting of the microphysics_type namelist.
984
985! if Kessler or Lin we can compute the fall velocity
986if (microphysics_type == 1 .or. microphysics_type == 2) then
987 call interpolate(state_handle, ens_size, location, qty_rainwater_mixing_ratio, qr, qr_istatus)
988 call track_status(ens_size, qr_istatus, precip_fall_speed, istatus, return_now)
989 if (return_now) return
990
991 if (microphysics_type == 2) then
992 call interpolate(state_handle, ens_size, location, qty_graupel_mixing_ratio, qg, qg_istatus)
993 call track_status(ens_size, qg_istatus, precip_fall_speed, istatus, return_now)
994 if (return_now) return
995
996 call interpolate(state_handle, ens_size, location, qty_snow_mixing_ratio, qs, qs_istatus)
997 call track_status(ens_size, qs_istatus, precip_fall_speed, istatus, return_now)
998 if (return_now) return
999
1000 endif
1001
1002 ! Done with Lin et al specific calls
1003 call interpolate(state_handle, ens_size, location, qty_density, rho, rho_istatus)
1004 call track_status(ens_size, rho_istatus, precip_fall_speed, istatus, return_now)
1005 if (return_now) return
1006
1007 call interpolate(state_handle, ens_size, location, qty_temperature, temp, temp_istatus)
1008 call track_status(ens_size, temp_istatus, precip_fall_speed, istatus, return_now)
1009 if (return_now) return
1010
1011
1012
1013 do imem = 1, ens_size
1014 if (istatus(imem) == 0) then
1015 call get_lk_precip_fall_speed(qr(imem), qg(imem), qs(imem), rho(imem), temp(imem), precip_fall_speed(imem))
1016 endif
1017 enddo
1018
1019 ! Done with Lin et al or Kessler -
1020
1021else if (microphysics_type == 5 .and. allow_dbztowt_conv) then
1022 ! Provided reflectivity field - will estimate fall velocity using empirical relations
1023 call get_expected_radar_ref(state_handle, ens_size, location, refl, refl_istatus)
1024 call track_status(ens_size, refl_istatus, precip_fall_speed, istatus, return_now)
1025 if (return_now) return
1026
1027 call interpolate(state_handle, ens_size, location, qty_density, rho, rho_istatus)
1028 call track_status(ens_size, rho_istatus, precip_fall_speed, istatus, return_now)
1029 if (return_now) return
1030
1031 do imem = 1, ens_size ! vectorize this routine?
1032 precip_fall_speed(imem) = dbztowt(refl(imem), rho(imem), missing_r8)
1033 enddo
1034
1035else if (microphysics_type < 0) then
1036 ! User requested setting fall velocity to zero - use with caution
1037 precip_fall_speed = 0.0_r8
1038 istatus(:) = 0
1039else
1040 ! Couldn't manage to compute a fall velocity
1041 precip_fall_speed = missing_r8
1042 istatus = 2
1043endif
1044
1045
1046end subroutine get_expected_fall_velocity

1077!----------------------------------------------------------------------
1078
1079subroutine get_expected_radar_ref(state_handle, ens_size, location, ref, istatus)
1080
1081! The main forward operator routine for radar reflectivity observations.
1082
1083type(ensemble_type), intent(in) :: state_handle
1084integer, intent(in) :: ens_size
1085type(location_type), intent(in) :: location
1086real(r8), intent(out) :: ref(ens_size)
1087integer, intent(out) :: istatus(ens_size)
1088
1089! Given a location and the state vector from one of the ensemble members,
1090! compute the model-predicted radar reflectivity that would be observed
1091! at that location. The returned value is in dBZ.
1092
1093! If apply_ref_limit_to_fwd_op, reflectivity values which are below
1094! reflectivity_limit_fwd_op will be set to lowest_reflectivity_fwd_op.
1095
1096! "interpolate()" ultimately calls model_mod routine "model_interpolate()"
1097! to get model values of qr, qg, qs, rho, and temp at the ob location. Then
1098! the routine "get_LK_reflectivity()" is called to compute the radar reflectivity
1099! factor value, Z, that corresponds to the hydrometeor and thermodynamic values.
1100
1101real(r8), dimension(ens_size) :: qr, qg, qs, rho, temp
1102integer, dimension(ens_size) :: qr_istatus, qg_istatus, qs_istatus
1103integer, dimension(ens_size) :: rho_istatus, temp_istatus
1104
1105logical, save :: first_time = .true.
1106integer :: imem
1107logical :: return_now
1108
1109if ( .not. module_initialized ) call initialize_module
1110
1111istatus(:) = 0
1112! Start with known values before calling interpolate routines.
1113qr(:) = 0.0_r8
1114qg(:) = 0.0_r8
1115qs(:) = 0.0_r8
1116rho(:) = 0.0_r8
1117temp(:) = 0.0_r8
1118
1119! If the model can return radar reflectivity data directly, give it a chance
1120! to do so. Otherwise, compute the various fields individually and then do
1121! the computation here. Note that the computation here is accurate only
1122! for the simple single-moment microphysics schemes (e.g., Kessler or Lin).
1123
1124! Try to draw from state vector first
1125call interpolate(state_handle, ens_size, location, qty_radar_reflectivity, ref, istatus)
1126
1127! If able to get value, QTY_RADAR_REFLECTIVITY is the
1128! the state so you can return here.
1129! Make sure to do apply_ref_limit_to_fwd_op check before "return"ing
1130if (any(istatus == 0) ) then
1131 if (apply_ref_limit_to_fwd_op) then
1132 where ((ref < reflectivity_limit_fwd_op) .and. (istatus == 0))
1133 ref = lowest_reflectivity_fwd_op
1134 end where
1135 endif
1136 return
1137endif
1138
1139! If the user explicitly wanted to interpolate in the field, try to complain
1140! if it could not. Note that the interp could fail for other reasons.
1141if (microphysics_type == 3 .or. microphysics_type == 5) then
1142 ! Could return a specific istatus code here to indicate this condition.
1143 if (first_time) then
1144 call error_handler(e_msg,'get_expected_radar_ref', &
1145 'interpolate failed. Reflectivity may NOT be in state vector', '', '', '')
1146 first_time = .false.
1147 endif
1148 return
1149endif
1150
1151istatus(:) = 0
1152
1153if (microphysics_type == 1 .or. microphysics_type == 2) then
1154
1155 call interpolate(state_handle, ens_size, location, qty_rainwater_mixing_ratio, qr, qr_istatus)
1156 call track_status(ens_size, qr_istatus, ref, istatus, return_now)
1157 if (return_now) return
1158
1159 if (microphysics_type == 2) then
1160 ! Also need some ice vars
1161 call interpolate(state_handle, ens_size, location, qty_graupel_mixing_ratio, qg, qg_istatus)
1162 call track_status(ens_size, qg_istatus, ref, istatus, return_now)
1163 if (return_now) return
1164
1165 call interpolate(state_handle, ens_size, location, qty_snow_mixing_ratio, qs, qs_istatus)
1166 call track_status(ens_size, qs_istatus, ref, istatus, return_now)
1167 if (return_now) return
1168
1169 endif
1170
1171 call interpolate(state_handle, ens_size, location, qty_density, rho, rho_istatus)
1172 call track_status(ens_size, rho_istatus, ref, istatus, return_now)
1173 if (return_now) return
1174
1175
1176 call interpolate(state_handle, ens_size, location, qty_temperature, temp, temp_istatus)
1177 call track_status(ens_size, temp_istatus, ref, istatus, return_now)
1178 if (return_now) return
1179
1180 do imem = 1, ens_size
1181 if (istatus(imem) == 0 ) then
1182 call get_lk_reflectivity(qr(imem), qg(imem), qs(imem), rho(imem), temp(imem), ref(imem))
1183 ! Always convert to dbz. Make sure the value, before taking the logarithm,
1184 ! is always slightly positive.
1185 ! tiny() is a fortran intrinsic function that is > 0 by a very small amount.
1186 ref(imem) = 10.0_r8 * log10(max(tiny(ref(imem)), ref(imem)))
1187 endif
1188 enddo
1189
1190else
1191 ! not in state vector and not Lin et al or Kessler so can't do reflectivity
1192 ref(:) = missing_r8
1193 istatus(:) = 1
1194endif
1195
1196if (apply_ref_limit_to_fwd_op) then
1197 where ((ref < reflectivity_limit_fwd_op) .and. (istatus == 0))
1198 ref = lowest_reflectivity_fwd_op
1199 end where
1200endif
1201

1211subroutine get_lk_reflectivity(qr, qg, qs, rho, temp, ref)
1212
1213! Computes the equivalent radar reflectivity factor in mm^6 m^-3 for
1214! simple single-moment microphysics schemes such as the Kessler and
1215! Lin et al. schemes. (Note: the equivalent reflectivity factor is usually
1216! called simply "reflectivity" throughout this code.) For more details
1217! about the computations, see the initialize_constants subroutine and
1218! the references below.
1219
1220real(r8), intent(in) :: qr, qg, qs ! rain,graupel,snow mixing ratios (kg kg^-1)
1221real(r8), intent(in) :: rho ! air density (kg m^-3)
1222real(r8), intent(in) :: temp ! air temperature (K)
1223real(r8), intent(out) :: ref ! equivalent reflectivity factor (mm^6 m^-3)
1224
1225! References: Ferrier, 1994 (JAS, 249-280)
1226! Gilmore et al., 2004 (MWR, 2610-2627)
1227! Lin et al., 1983 (JCAM, 1065-1092)
1228! Smith 1984 (JCAM, 1258-1260)
1229! Smith at al., 1975 (JAM, 1156-1165)
1230! Caya, "Radar observations in DART" (DART subversion repository)
1231
1232real(r8) :: precip
1233
1234if ( .not. module_initialized ) call initialize_module
1235
1236ref = 0.0_r8
1237
1238! RAIN
1239if ( qr >= 1.0e-6_r8 ) then
1240 precip = rho * qr
1241 ref = ref + param_refl_r * (precip**1.75_r8)
1242endif
1243
1244! HAIL / GRAUPEL
1245! The exponent 1.6625 is 1.75*0.95. The 0.95 factor is included as an
1246! approximation for Mie scattering (Smith et al. 1975). This approximation
1247! is appropriate for a 10-cm wavelength radar.
1248
1249if ( qg >= 1.0e-6_r8 ) then
1250 precip = rho * qg
1251 if (.not. allow_wet_graupel .or. temp < 273.15_r8) then
1252 ref = ref + param_refl_dry_g * (precip**1.75_r8)
1253 else
1254 ref = ref + param_refl_wet_g * (precip**1.6625_r8)
1255 endif
1256endif
1257
1258! SNOW
1259if ( qs >= 1.0e-6_r8 ) then
1260 precip = rho * qs
1261 if ( temp < 273.15_r8 ) then
1262 ref = ref + param_refl_dry_s * (precip**1.75_r8)
1263 else
1264 ref = ref + param_refl_wet_s * (precip**1.75_r8)
1265 endif
1266endif

1272subroutine get_lk_precip_fall_speed(qr, qg, qs, rho, temp, precip_fall_speed)
1273
1274! Computes power-weighted precipitation fall speed in m s^-1 for simple single-
1275! moment microphysics schemes such as the Kessler and Lin et al. schemes.
1276! For more details, see the initialize_constants subroutine and the
1277! references below.
1278
1279real(r8), intent(in) :: qr, qg, qs ! rain,graupel,snow mixing ratios (kg kg^-1)
1280real(r8), intent(in) :: rho ! air density (kg m^-3)
1281real(r8), intent(in) :: temp ! air temperature (K)
1282real(r8), intent(out) :: precip_fall_speed ! power-weighted precip. fall speed (m s^-1)
1283
1284! References: Ferrier, 1994 (JAS, 249-280)
1285! Gilmore et al., 2004 (MWR, 2610-2627)
1286! Lin et al., 1983 (JCAM, 1065-1092)
1287! Smith 1984 (JCAM, 1258-1260)
1288! Smith at al., 1975 (JAM, 1156-1165)
1289! Caya, "Radar observations in DART" (DART subversion repository)
1290
1291! alpha is the adjustment for air density in the emperical
1292! dropsize-fall speed equation
1293real(r8) :: precip_r, precip_s, precip_g, alpha, ref
1294
1295if ( .not. module_initialized ) call initialize_module
1296
1297precip_r = rho * qr
1298precip_s = rho * qs
1299precip_g = rho * qg
1300alpha = sqrt(param_rhos0/rho)
1301
1302precip_fall_speed = 0.0_r8
1303
1304! RAIN
1305if (qr >= 1.0e-6_r8) then
1306 precip_fall_speed = precip_fall_speed + alpha * param_fs_r * (precip_r**param_powr)
1307endif
1308
1309! HAIL/GRAUPEL
1310if (qg >= 1.0e-6_r8) then
1311 if (.not. allow_wet_graupel .or. temp < 273.15_r8) then
1312 precip_fall_speed = precip_fall_speed + sqrt(param_e/rho) * &
1313 param_fs_dry_g * (precip_g**param_powg_dry)
1314 else
1315 precip_fall_speed = precip_fall_speed + sqrt(param_e/rho) * &
1316 param_fs_wet_g * ((precip_g/(pi*rho_graupel))**param_powg_wet)
1317 endif
1318endif
1319
1320! SNOW
1321if (qs >= 1.0e-6_r8) then
1322 if ( temp < 273.15_r8 ) then
1323 precip_fall_speed = precip_fall_speed + alpha * param_fs_dry_s * &
1324 (precip_s**param_pows)
1325 else
1326 precip_fall_speed = precip_fall_speed + alpha * param_fs_wet_s * &
1327 (precip_s**param_pows)
1328 endif
1329endif
1330
1331if (precip_fall_speed > 0.0_r8) then
1332 call get_lk_reflectivity(qr, qg, qs, rho, temp, ref)
1333 if (ref > 0.0_r8) then
1334 precip_fall_speed = precip_fall_speed/ref
1335 else
1336 precip_fall_speed = 0.0_r8
1337 endif
1338endif

1340end subroutine get_lk_precip_fall_speed

1370subroutine initialize_constants()
1371
1372! Initialize module global constants.
1373
1374! IMPORTANT: Uses namelist values, so this routine cannot be called until
1375! after the namelist has been read.
1376
1377! NOTE: gravity is currently hardcoded here. We should find a way to let the
1378! model have input if it uses a slightly different value for G, or if it is
1379! working on a different planet with an entirely different set of constants.
1380! Question: how much impact on the results does changing G have?
1381!
1382! Most of the constants below are used for the computation of reflectivity and
1383! power-weighted precipitation fall speed from the model state. Only simple
1384! single-moment microphysics schemes, such as the Kessler and Lin et al.
1385! schemes, are currently supported. For these schemes, rain, snow, and
1386! graupel/hail are assumed to have inverse exponential size distributions:
1387!
1388! n(D)=n0*exp(-lambda*D),
1389!
1390! where D is the particle diameter, n is the number of particles per unit
1391! volume and per particle size interval, n0 is the intercept parameter, and
1392! lambda is the slope parameter. Lambda is a function of the model-predicted
1393! hydrometeor mixing ratio and air density.
1394!
1395! For Rayleigh scattering from a spherical raindrop, the reflectivity is
1396! proportional to the 6th moment of the raindrop's diameter. The basic form of
1397! the complicated expressions below comes from integrating this relationship
1398! between reflectivity and diameter over the entire distribution of hydrometeor
1399! sizes. Complications arise because not all hydrometeors are spherical, and
1400! returned power depends significantly on whether hydrometeors have a wet or
1401! dry surface. In addition, for large hailstones, Mie scattering rather than
1402! Rayleigh scattering must be considered.
1403!
1404! References: Ferrier, 1994 (JAS, 249-280)
1405! Gilmore et al., 2004 (MWR, 2610-2627)
1406! Lin et al., 1983 (JCAM, 1065-1092)
1407! Smith 1984 (JCAM, 1258-1260)
1408! Smith at al., 1975 (JAM, 1156-1165)
1409! Caya, "Radar observations in DART" (DART subversion repository)
1410
1411
1412! In general, the value used for gravity here should match the value used
1413! for gravity in the model.
1414param_gravity = 9.81_r8 ! Acceleration of gravity (m s^-2)
1415param_a = 8.42e20_r8
1416param_b = 0.8_r8
1417param_c = 4.84e18_r8
1418param_d = 0.25_r8
1419param_cd = 0.6_r8
1420param_rhos0 = 1.0_r8
1421param_e = 4.0_r8*param_gravity*rho_graupel/(3.0_r8*param_cd)
1422
1423param_gam7b = 3376.92_r8
1424param_gam7d = 1155.38_r8
1425param_gam7f = 1871.25_r8
1426param_powr = (7.0_r8 + param_b)/4.0_r8
1427param_pows = (7.0_r8 + param_d)/4.0_r8
1428param_powg_dry = (7.0_r8 + 0.5_r8) /4.0_r8
1429param_powg_wet = 1.7875_r8
1430
1431
1432! In the expressions for power-weighted precipitation fall speed for each
1433! hydrometeor category, the following parameters are computed from the
1434! constants that do not vary in time and space. These equations are rather
1435! complicated, so refer to "Radar observations in DART" by A. Caya for details.
1436! (A pdf of this paper is in the DART subversion repository.)
1437! The 0.95 exponenent in the equation for param_fs_wet_g is an approximation
1438! for the effects of Mie scattering (Smith et al. 1975). This approximation
1439! is appropriate for a 10-cm wavelength radar.
1440
1441param_fs_r = n0_rain * param_a * param_gam7b / &
1442 (pi * rho_rain * n0_rain)**param_powr
1443
1444param_fs_wet_s = n0_snow * param_c * param_gam7d / &
1445 (pi * rho_snow * n0_snow)**param_pows
1446
1447param_fs_dry_s = dielectric_factor * ((rho_snow/rho_rain)**2.0_r8) * &
1448 param_fs_wet_s
1449
1450param_fs_wet_g = ((7.2e20_r8)**0.95_r8) * param_gam7f / &
1451 (720.0_r8 * (n0_graupel**0.8375_r8))
1452
1453param_fs_dry_g = 1.0e18_r8 * dielectric_factor * &
1454 ((rho_graupel/rho_rain)**2.0_r8) * n0_graupel * param_gam7f / &
1455 (pi * rho_graupel * n0_graupel)**param_powg_dry
1456
1457! In the expressions for reflectivity for each hydrometeor category, the
1458! following parameters are computed from the constants that do not vary in time
1459! and space. Computing these parameters now means that the equations in the
1460! get_LK_reflectivity subroutine will have the following simple form:
1461!
1462! ref = param_refl_r * (rho*qr)**1.75.
1463!
1464! The 0.95 exponenent in the equation for param_refl_wet_g is an approximation
1465! for the effects of Mie scattering (Smith et al. 1975). This approximation is
1466! appropriate for a 10-cm wavelength radar.
1467
1468param_refl_r = 7.2e20_r8 / (((pi*rho_rain)**1.75_r8)*(n0_rain**0.75_r8))
1469
1470param_refl_wet_s = 7.2e20_r8 / (((pi*rho_snow)**1.75_r8)*(n0_snow**0.75_r8))
1471
1472param_refl_dry_s = dielectric_factor * ((rho_snow/rho_rain)**2.0_r8) * &
1473 param_refl_wet_s
1474
1475param_refl_wet_g = (7.2e20_r8/(((pi*rho_graupel)**1.75_r8) * &
1476 (n0_graupel**0.75_r8)))**0.95_r8
1477
1478param_refl_dry_g = dielectric_factor * ((rho_graupel/rho_rain)**2.0_r8) * &
1479 7.2e20_r8 / (((pi*rho_graupel)**1.75_r8)*(n0_graupel**0.75_r8))
1480
1481
1482
1483end subroutine initialize_constants
